üõ° How is this 100% secure?

Think of it like this: We manufacture the lockbox, but you create the key.
When you click "Create," your browser generates a random key and locks the message locally. We store the locked box, but the key is only ever shown to you (in the link). We physically cannot open your message because we never see the key.

  • Algorithm: AES-GCM (256-bit) via native Web Crypto API.
  • Key Storage: The key is passed in the URL Hash Fragment (`#`). Browsers never send the hash to the server.
  • Zero Knowledge: The server receives only the encrypted blob and the IV. It has zero computational way to decrypt the data.
This frontend is written in plain, non-obfuscated JavaScript. Check out the source code and see the client-side encryption for yourself.
